# BiteBlog 接口说明文档

## 1. 统一响应格式

所有接口返回统一 JSON 格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": { ... },
  "timestamp": "2026-05-08T15:42:19+08:00",
  "requestId": "trace-uuid"
}
```

### 错误码

| Code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录或 Token 过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 请求频率超限 |
| 500 | 系统内部错误 |
| 1001 | 用户不存在 |
| 1002 | 手机号已注册 |
| 1003 | 密码错误 |
| 1004 | 账号已被禁用 |
| 2001 | 笔记不存在或已删除 |
| 2002 | 图片上传失败 |
| 2003 | 笔记内容不合法 |
| 3001 | Feed 无新内容 |
| 4001 | 推荐服务暂不可用 |
| 5001 | 地理位置获取失败 |
| 6001 | 通知发送失败 |

## 2. 鉴权方式

- 登录成功后返回 JWT Token
- 需要鉴权的接口在请求头携带：`Authorization: Bearer <token>`
- Gateway 统一校验 Token，将解析后的 `userId` 通过 `X-User-Id` 请求头传递给下游服务
- 白名单接口（无需 Token）：`/user/login`、`/user/register`、`/rank/top10`

## 3. 分页约定

### 游标分页（Feed / 推荐）

```
GET /api/feed/timeline?cursor=1714972800000&size=20
```

响应：

```json
{
  "code": 200,
  "data": {
    "list": [...],
    "cursor": "1714972700000",
    "hasMore": true
  }
}
```

### 传统分页（评论 / 排行 / 通知）

```
GET /api/post/1/comments?page=1&size=20
```

响应：

```json
{
  "code": 200,
  "data": {
    "list": [...],
    "total": 156
  }
}
```

## 4. 网关路由规则

```
/api/user/**       → user-service     (8081)
/api/post/**       → post-service     (8082)
/api/feed/**       → feed-service     (8083)
/api/recommend/**  → recommend-service (8084)
/api/location/**   → location-service (8085)
/api/rank/**       → rank-service     (8086)
/api/notify/**     → notify-service   (8087)
```

Gateway 自动 StripPrefix `/api`，下游服务看到的是 `/user/**`、`/post/**` 等。

## 5. 已实现接口

### 5.1 用户服务 (user-service) — **已完成**

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| POST | `/user/register` | 否 | 用户注册 |
| POST | `/user/login` | 否 | 用户登录 |
| GET | `/user/{id}` | 是 | 获取用户主页 |
| POST | `/user/follow/{id}` | 是 | 关注 / 取关（幂等切换） |
| GET | `/user/{id}/following` | 是 | 获取关注列表（传统分页） |
| GET | `/user/{id}/followers` | 是 | 获取粉丝列表（传统分页） |

#### POST /user/register

注册新用户，手机号和用户名必须唯一。

请求体：

```json
{
  "phone": "13800000001",
  "password": "12345678",
  "username": "美食达人"
}
```

成功响应 (code=200)：

```json
{
  "code": 200,
  "data": {
    "userId": 1,
    "token": "eyJhbGciOiJIUzM4NCJ9..."
  }
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 手机号已注册 | 1002 | 手机号已注册 |
| 用户名已被使用 | 400 | 用户名已被使用 |

#### POST /user/login

验证手机号和密码，返回 JWT Token。

请求体：

```json
{
  "phone": "13800000001",
  "password": "12345678"
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzM4NCJ9...",
    "userId": 1,
    "username": "美食达人"
  }
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 用户不存在 | 1001 | 用户不存在 |
| 密码错误 | 1003 | 密码错误 |

#### GET /user/{id}

获取用户主页信息，Redis 缓存优先（TTL 2 小时）。

请求头：`Authorization: Bearer <token>`

成功响应：

```json
{
  "code": 200,
  "data": {
    "userId": 1,
    "username": "美食达人",
    "avatar": null,
    "bio": "专注探店",
    "followerCount": 10,
    "followingCount": 5,
    "likeCount": 100,
    "isBigV": false,
    "createdAt": "2026-05-08T15:00:00"
  }
}
```

#### POST /user/follow/{id}

关注或取关目标用户（幂等操作：已关注则取关，未关注则关注）。

请求头：`Authorization: Bearer <token>`、`X-User-Id: <当前用户ID>`

成功响应：

```json
{
  "code": 200,
  "data": {
    "followed": true
  }
}
```

`followed: true` 表示关注成功，`followed: false` 表示取关成功。

#### GET /user/{id}/following

获取用户的关注列表，传统分页。

请求头：`Authorization: Bearer <token>`

参数：`page`（默认 1）、`size`（默认 20）

成功响应：

```json
{
  "code": 200,
  "data": {
    "list": [
      { "userId": 2, "username": "吃货老张", "avatar": null, "bio": "..." },
      { "userId": 3, "username": "探店小王", "avatar": null, "bio": "..." }
    ],
    "total": 15
  }
}
```

#### GET /user/{id}/followers

获取用户的粉丝列表，传统分页。参数和响应格式与关注列表相同。
