# BiteBlog 接口说明文档

## 1. 统一响应格式

所有接口返回统一 JSON 格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": { ... },
  "timestamp": "2026-05-08T15:42:19+08:00",
  "requestId": "trace-uuid"
}
```

### 错误码

| Code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录或 Token 过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 请求频率超限 |
| 500 | 系统内部错误 |
| 1001 | 用户不存在 |
| 1002 | 手机号已注册 |
| 1003 | 密码错误 |
| 1004 | 账号已被禁用 |
| 2001 | 笔记不存在或已删除 |
| 2002 | 图片上传失败 |
| 2003 | 笔记内容不合法 |
| 3001 | Feed 无新内容 |
| 4001 | 推荐服务暂不可用 |
| 5001 | 地理位置获取失败 |
| 6001 | 通知发送失败 |

## 2. 鉴权方式

- 登录成功后返回 JWT Token
- 需要鉴权的接口在请求头携带：`Authorization: Bearer <token>`
- Gateway 统一校验 Token，将解析后的 `userId` 通过 `X-User-Id` 请求头传递给下游服务
- 白名单接口（无需 Token）：`/user/login`、`/user/register`、`/rank/top10`

## 3. 分页约定

### 游标分页（Feed / 推荐）

```
GET /api/feed/timeline?cursor=1714972800000&size=20
```

响应：

```json
{
  "code": 200,
  "data": {
    "list": [...],
    "cursor": "1714972700000",
    "hasMore": true
  }
}
```

### 传统分页（评论 / 排行 / 通知）

```
GET /api/post/1/comments?page=1&size=20
```

响应：

```json
{
  "code": 200,
  "data": {
    "list": [...],
    "total": 156
  }
}
```

## 4. 网关路由规则

```
/api/user/**       → user-service     (8081)
/api/post/**       → post-service     (8082)
/api/feed/**       → feed-service     (8083)
/api/recommend/**  → recommend-service (8084)
/api/location/**   → location-service (8085)
/api/rank/**       → rank-service     (8086)
/api/notify/**     → notify-service   (8087)
```

Gateway 自动 StripPrefix `/api`，下游服务看到的是 `/user/**`、`/post/**` 等。

## 5. 已实现接口

### 5.1 用户服务 (user-service) — **已完成**

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| POST | `/user/register` | 否 | 用户注册 |
| POST | `/user/login` | 否 | 用户登录 |
| GET | `/user/{id}` | 是 | 获取用户主页 |
| POST | `/user/follow/{id}` | 是 | 关注 / 取关（幂等切换） |
| GET | `/user/{id}/following` | 是 | 获取关注列表（传统分页） |
| GET | `/user/{id}/followers` | 是 | 获取粉丝列表（传统分页） |

#### POST /user/register

注册新用户，手机号和用户名必须唯一。

请求体：

```json
{
  "phone": "13800000001",
  "password": "12345678",
  "username": "美食达人"
}
```

成功响应 (code=200)：

```json
{
  "code": 200,
  "data": {
    "userId": 1,
    "token": "eyJhbGciOiJIUzM4NCJ9..."
  }
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 手机号已注册 | 1002 | 手机号已注册 |
| 用户名已被使用 | 400 | 用户名已被使用 |

#### POST /user/login

验证手机号和密码，返回 JWT Token。

请求体：

```json
{
  "phone": "13800000001",
  "password": "12345678"
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzM4NCJ9...",
    "userId": 1,
    "username": "美食达人"
  }
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 用户不存在 | 1001 | 用户不存在 |
| 密码错误 | 1003 | 密码错误 |

#### GET /user/{id}

获取用户主页信息，Redis 缓存优先（TTL 2 小时）。

请求头：`Authorization: Bearer <token>`

成功响应：

```json
{
  "code": 200,
  "data": {
    "userId": 1,
    "username": "美食达人",
    "avatar": null,
    "bio": "专注探店",
    "followerCount": 10,
    "followingCount": 5,
    "likeCount": 100,
    "isBigV": false,
    "createdAt": "2026-05-08T15:00:00"
  }
}
```

#### POST /user/follow/{id}

关注或取关目标用户（幂等操作：已关注则取关，未关注则关注）。

请求头：`Authorization: Bearer <token>`、`X-User-Id: <当前用户ID>`

成功响应：

```json
{
  "code": 200,
  "data": {
    "followed": true
  }
}
```

`followed: true` 表示关注成功，`followed: false` 表示取关成功。

#### GET /user/{id}/following

获取用户的关注列表，传统分页。

请求头：`Authorization: Bearer <token>`

参数：`page`（默认 1）、`size`（默认 20）

成功响应：

```json
{
  "code": 200,
  "data": {
    "list": [
      { "userId": 2, "username": "吃货老张", "avatar": null, "bio": "..." },
      { "userId": 3, "username": "探店小王", "avatar": null, "bio": "..." }
    ],
    "total": 15
  }
}
```

#### GET /user/{id}/followers

获取用户的粉丝列表，传统分页。参数和响应格式与关注列表相同。

### 5.2 笔记服务 (post-service) — **已完成**

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| POST | `/post/upload-image` | 是 | 上传图片到 MinIO |
| POST | `/post/publish` | 是 | 发布笔记 |
| GET | `/post/{id}` | 可选 | 笔记详情 |
| POST | `/post/{id}/like` | 是 | 点赞 / 取消点赞（幂等） |
| POST | `/post/{id}/favorite` | 是 | 收藏 / 取消收藏（幂等） |
| POST | `/post/{id}/comment` | 是 | 发表评论 |
| GET | `/post/{id}/comments` | 可选 | 评论列表（传统分页） |
| DELETE | `/post/{id}` | 是 | 删除笔记（仅作者） |
| GET | `/post/search` | 可选 | ES 全文搜索笔记 |

#### POST /post/upload-image

上传单张图片到 MinIO，返回可访问 URL。前端先调此接口上传图片，拿到 URL 后再调 `/post/publish` 发布笔记。

请求：`multipart/form-data`，字段名 `file`

成功响应：

```json
{
  "code": 200,
  "data": "http://localhost:9000/biteblog-post-images/post/uuid_xxx.jpg"
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 上传失败 | 2002 | 图片上传失败 |

#### POST /post/publish

发布探店笔记。图片 URL 需要先通过 `/post/upload-image` 获取。

请求头：`X-User-Id: <当前用户ID>`

请求体：

```json
{
  "title": "测试探店：老王烧烤",
  "content": "味道非常好，推荐烤羊排和大拉皮",
  "shopName": "老王烧烤",
  "address": "武汉市洪山区珞喻路1037号",
  "longitude": 114.35,
  "latitude": 30.52,
  "scoreColor": 4,
  "scoreSmell": 3,
  "scoreTaste": 5,
  "imageUrls": ["http://localhost:9000/biteblog-post-images/post/xxx.jpg"]
}
```

成功响应：

```json
{
  "code": 200,
  "data": { "postId": 1 }
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 标题为空 | 400 | title: 标题不能为空 |
| 内容为空 | 400 | content: 内容不能为空 |

> 发布成功后异步写入 ES post_index 并发送 RabbitMQ `note.published` 事件（Feed 分发 + 排行更新 + 地理位置写入）。

#### GET /post/{id}

获取笔记详情。未登录也可访问（`X-User-Id` 可选），但 `liked`/`favorited` 字段为 false。

请求头：`X-User-Id: <当前用户ID>`（可选）

成功响应：

```json
{
  "code": 200,
  "data": {
    "postId": 1,
    "authorId": 1,
    "title": "测试探店：老王烧烤",
    "content": "味道非常好，推荐烤羊排和大拉皮",
    "shopName": "老王烧烤",
    "address": "武汉市洪山区珞喻路1037号",
    "longitude": 114.35,
    "latitude": 30.52,
    "imageUrls": ["http://localhost:9000/biteblog-post-images/post/xxx.jpg"],
    "scoreColor": 4,
    "scoreSmell": 3,
    "scoreTaste": 5,
    "likeCount": 1,
    "collectCount": 2,
    "commentCount": 3,
    "liked": true,
    "favorited": false,
    "createdAt": "2026-05-09T10:30:00"
  }
}
```

> Redis 缓存优先（key=`post:cache:{postId}`，TTL 30min），缓存未命中回源 MySQL 并写回缓存。

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 笔记不存在或已删除 | 2001 | 笔记不存在或已删除 |

#### POST /post/{id}/like

点赞或取消点赞（幂等切换：已赞则取消，未赞则点赞）。

请求头：`X-User-Id: <当前用户ID>`

成功响应：

```json
{
  "code": 200,
  "data": { "liked": true }
}
```

`liked: true` 表示点赞成功，`liked: false` 表示取消点赞成功。

> 并发安全：`note_like` 表 `UNIQUE KEY uk_like(note_id, user_id)` 保证不重复。计数值通过 `SET like_count = like_count + 1` 原子更新。点赞后发送 RabbitMQ 事件通知排行服务和通知服务。

#### POST /post/{id}/favorite

收藏或取消收藏（幂等切换），与点赞逻辑一致。

请求头：`X-User-Id: <当前用户ID>`

成功响应：

```json
{
  "code": 200,
  "data": { "favorited": true }
}
```

#### POST /post/{id}/comment

发表评论或回复。`parentId` 为空表示顶级评论，非空表示回复某条评论。

请求头：`X-User-Id: <当前用户ID>`

请求体：

```json
{
  "content": "写得真好，周末就去试试",
  "parentId": null
}
```

成功响应：

```json
{
  "code": 200,
  "data": { "commentId": 1 }
}
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 评论内容为空 | 400 | content: 评论内容不能为空 |
| 笔记不存在 | 2001 | 笔记不存在或已删除 |

#### GET /post/{id}/comments

获取笔记的评论列表，传统分页。仅返回顶级评论（`parent_id IS NULL`），每个顶级评论嵌套其回复列表。

参数：`page`（默认 1）、`size`（默认 20）

成功响应：

```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "commentId": 1,
        "userId": 2,
        "content": "写得真好，周末就去试试",
        "parentId": null,
        "createdAt": "2026-05-09T10:31:00",
        "replies": [
          {
            "commentId": 2,
            "userId": 1,
            "content": "谢谢支持！记得点烤羊排",
            "parentId": 1,
            "createdAt": "2026-05-09T10:32:00",
            "replies": null
          }
        ]
      }
    ],
    "total": 5
  }
}
```

#### DELETE /post/{id}

删除笔记（仅作者可删）。逻辑删除，`status` 置为 0。

请求头：`X-User-Id: <当前用户ID>`

成功响应：

```json
{ "code": 200 }
```

异常响应：

| 情况 | code | msg |
|------|------|-----|
| 笔记不存在 | 2001 | 笔记不存在或已删除 |
| 非作者删除 | 403 | 无权限操作 |

> 删除后清除 Redis 缓存、异步删除 ES 索引、发送 RabbitMQ `note.deleted` 事件通知 Feed 服务清理。

#### GET /post/search

ES 全文搜索笔记，对标题、内容、店铺名进行 IK 分词匹配。

参数：`keyword`（必填）、`page`（默认 1）、`size`（默认 20）

成功响应：

```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "postId": "1",
        "title": "测试探店：老王烧烤",
        "shopName": "老王烧烤",
        "cover": "http://localhost:9000/biteblog-post-images/post/xxx.jpg",
        "likeCount": 1
      }
    ],
    "total": 1
  }
}
```

> ES 不可用时降级返回空列表（`total: 0`），不报 500。
