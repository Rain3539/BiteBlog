### 生成的数据

| 数据类型 | 内容 | 说明 |
|----------|------|------|

| 互动记录 | `13800000004` 对上述笔记点赞、收藏、评论各 1 次 | 触发 Post 服务向 `biteblog.interaction` 发送 MQ 消息 |
| notification 记录 | `receiver_id=bb_bigv_01.userId`，类型为 like/collect/comment | Notify 消费 MQ 后写入，预期 ≥ 3 条 |

### 跨服务依赖关系

```
init-notify-data.ps1
  ├── 登录 13800000001 / 13800000004 (user-service 8081)
  ├── 13800000001 发布笔记 (post-service 8082 via gateway 8080)
  ├── 13800000004 点赞/收藏/评论 → Post 发 MQ (biteblog.interaction)
  │     └── Notify 消费 → 写 notification 表 → WS 推送
  └── 验证 /notify/list total≥3 + /notify/unread-count (notify-service 8087)
```

### 执行说明

```powershell
cd BiteBlog\sql
.\init-notify-data.ps1
```

预期输出：`[OK] /notify/list total=3`（或更多）和 `[OK] /notify/unread-count unreadCount=3`（或更多）。

结果自动保存至 `sql/notify-init-result.txt`。

### 自操作过滤验证

脚本还会让 `13800000001` 对自己的笔记点赞，Notify 应过滤（`authorId == userId`），不写入通知，可通过比对 `notification` 表中 `sender_id = 13800000001 userId` 的条数来验证。

### Redis 数据

脚本会写入：

| Key | 类型 | 内容 |
|-----|------|------|
| `recommend:hot:pool` | ZSet | 60 条推荐样例笔记，score 从 300 递减 |
| `rank:daily:{yyyy-MM-dd}` | ZSet | 同步写入 60 条推荐样例，供 Recommend 冷启动优先复用 Rank 日榜 |
| `behavior:{userId}` | String/JSON | 推荐请求后生成的用户近期行为画像缓存，TTL 分钟级 |
| `recommend:itemcf:similar:{postId}` | ZSet | 部分样例笔记的预计算相似笔记，用作 ItemCF 的 Redis 降级数据 |
| `exposure:{13800000005的userId}` | Set | 预置 foodie 用户已曝光的第一条推荐样例 |

### Elasticsearch 数据

脚本会尝试把 60 条 `Recommend Test` 样例同步到 ES `post_index`，并把 ItemCF 相似关系同步到 ES `item_sim_index`：

| 索引 | 内容 | 说明 |
|------|------|------|
| `post_index` | `postId/title/content/store_name/shopName/tags/status/like_count/...` | ES 可用时，`tag` 和 `city` 查询优先走该索引 |
| `item_sim_index` | `item_id/similar_item_id/score` | ES 可用时，ItemCF 优先按用户交互过的 `item_id` 查询相似笔记 |

如果 ES 未启动或索引未创建，脚本会输出 warning，但 MySQL 和 Redis 数据仍会继续初始化，Recommend Service 会自动降级为 MySQL 关键词召回。

### 数据用途

| 测试项 | 依赖数据 | 说明 |
|--------|----------|------|
| 冷启动推荐 | `13800000060` + `recommend:hot:pool` | 新用户请求 `/recommend/discover` 时返回近期热门 |
| 标签推荐 | `tag=Hotpot` / `tag=Tea` | 优先通过 ES `post_index` 召回，失败时 MySQL 兜底 |
| 预计算 ItemCF | ES `item_sim_index` | ES 相似索引中的额外笔记进入候选 |
| ItemCF 降级 | `recommend:itemcf:similar:{postId}` / 共同行为 | ES 不可用或无结果时，降级 Redis 相似池，再降级在线行为扫描 |
| 曝光去重 | `exposure:{userId}` | Redis Lua 返回前原子预占，已曝光笔记再次推荐时被过滤 |
| 前端发现页 | 60 条推荐样例 + 曝光上报 | `/discover` 页面可展示推荐卡片、加载更多并上报曝光 |
| MQ 联调 | Post Service 发帖/互动事件 | Recommend 消费 `note.published`、`note.deleted`、`interaction.*`，刷新 ES `post_index`、ES `item_sim_index` 和 Redis `recommend:hot:pool` |

### 执行说明

先确保已执行基础用户脚本：

```powershell
.\sql\init-data.ps1
```

再执行推荐数据脚本：

```powershell
.\sql\init-recommend-data.ps1
```

如果脚本提示缺少用户，说明基础用户数据尚未准备好，需要先启动 User Service 并执行 `init-data.ps1`。
